## Files touched

- `LogUpload/ChunkUploader.cs` — added `internal static string BuildUrl(string baseUrl, string region, long levelUuid)`
  (`{baseUrl}/run/{region}/{levelUuid}/events`); `UploadChunksFireAndForget` and
  `UploadSequentialAsync` gained a `string region` parameter (after `baseUrl`); updated the
  three doc comments that spelled the old (region-less) URL shape.
- `LogUpload/LogUploader.cs` — `PostSupplementAsync`'s URL now reads
  `{ApiBase}/run/{log.Header.Region}/{levelUuid}/supplement`; updated the `ApiBase` doc comment.
- `Replay/PositionUploader.cs` — `UploadFireAndForget` gained a `string region` first parameter;
  URL built via `string.Concat(BaseUrl, region, "/", levelUuid, "/positions")` (brief's exact
  shape — `BaseUrl` already ends in `run/`); updated the endpoint doc comment.
- `LogUpload/PortraitReport.cs` — `WriteBody` gained a trailing `string region` parameter, emitted
  as `,"region":"{region}"` immediately after `localUid` and before `nonce`/`sig`/`entries` — i.e.
  outside the signed `entries` array. Left the CROSS-REPO INVARIANT comment (entries' field list:
  uid/profileUrl/halfbodyUrl/identity{...}) completely untouched, since `region` is a top-level
  sibling of `localUid`/`nonce`/`sig`, not part of `entries`. `CanonicalPayload` untouched (no
  region parameter, as required).
- `Plugin.PortraitUpload.cs` — two `WriteBody` call sites now pass
  `_services.GameEnvironment.RegionCode`:
  - `MaybeReportPortraits()` — added `if (!RegionKnownOrWarn()) return;` at the top (skips the
    roster batch entirely on unknown region), per the brief.
  - `SendSelfMasterScoreEntry(...)` — **not** re-gated on `RegionKnownOrWarn()` (see inline comment
    added at the call site): its sole caller, `RefreshAndSendSelfMasterScore()`, is only invoked
    from `AssembleAndUpload` in `Plugin.LogUpload.cs`, which already passed the same gate before
    reaching that line — and `RefreshAndSendSelfMasterScore` unconditionally persists
    `lastSentKey = score` right after calling `SendSelfMasterScoreEntry`. Adding an unreachable
    early-return there would create a latent bug (marking the score "sent" when a future,
    differently-gated call path silently skipped it) for zero present-day behavior change. Only
    the required arity fix (pass region) was applied.
- `Plugin.LogUpload.cs`:
  - `AssembleAndUpload`'s web run URL: `"https://logs.stellarresonance.app/run/" + log.Header.Region + "/" + levelUuid`.
  - `OnSummaryUploadOk`'s `ChunkUploader.UploadChunksFireAndForget(...)` call now passes
    `log.Header.Region` (after `LogUploader.ApiBase`, before `levelUuid`).
- `Plugin.Replay.cs` — `UploadReplayDoc(PositionUploadDoc doc)`'s
  `PositionUploader.UploadFireAndForget(...)` call now passes `_services.GameEnvironment.RegionCode`
  as the new first argument. Documented why: `PositionUploadDoc` carries no region field of its
  own, and neither of `UploadReplayDoc`'s two callers (the summary-upload callback legs in
  `Plugin.LogUpload.cs`'s `OnSummaryUploadOk`/`OnSummaryUploadFailed`, and the no-summary-fired
  fallback in `Plugin.History.cs` line 142) has a `CombatLog` in scope — only `replayDoc`/`entry` —
  so `log.Header.Region` isn't reachable there; `GameEnvironment.RegionCode` is the only source,
  and per `IGameEnvironment`'s doc comment ("latched at boot and never changes during a session")
  it is value-identical to what `log.Header.Region` would have held anyway.
  Positions upload is **not** newly gated on `RegionKnownOrWarn()` — this matches its pre-existing
  "always try, even when auto-upload is off" posture (see the existing
  `OnSummaryUploadFailed`/`Plugin.History.cs` fallback comments); the brief's Step 5 only asks for
  the region argument to be threaded through, not a new gate. Worst case on a genuinely unknown
  region: the worker's `parseRegion` (only `"sea"`/`"jp"` allowlisted) 400s the POST, which the
  existing fire-and-forget error path already logs and swallows — no crash, no regression.
